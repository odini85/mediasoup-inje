# 구조 리팩토링

## 방향

- 기존 코드는 놔두고 별도로 진행한다.
- server/client 환경을 별도로 분리한다.
- room/user/peer manager 분리
  - 서버 입장에서 모든 참조를 들고 있다며 좋지만, room, peer, user에 대한 관계설정이 필요함
  - room/peer/user manager를 만들어서 관리
- peer
  - 종류별 송/수신 구현체를 분리한다.
    - 이 구현체는 peer가 소유한다.
    - 예 : inje-mediasoup.js

## 용어 정리

- VO
  - Server <-> Client 주고 받는 데이터
